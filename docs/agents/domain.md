# Domain Docs

This repository uses a single domain context.

## Before exploring

- Read `CONTEXT.md`.
- Read relevant ADRs under `docs/adr/`.
- If either location is absent, proceed silently.

## Vocabulary

Use domain terms defined in `CONTEXT.md`. Avoid introducing synonyms for existing concepts.

## ADR conflicts

Explicitly identify proposals that conflict with an existing ADR rather than silently overriding the decision.
